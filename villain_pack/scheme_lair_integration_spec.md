# Scheme And Lair Integration

These are the next two doctrine buckets feeding the villain world graph.

## Schemes

Schemes are not just plots.
They are reusable villain action grammars:
- heists
- coverups
- reversals
- betrayals
- hostage pageants
- inheritance traps
- research loops

A scheme should answer:
- what the villain is trying to do
- who gets exposed to it
- where the betrayal surface lives
- what the twist shape is
- what breaks the plan

## Lairs

Lairs are not just places.
They are operating environments with architecture, access logic, hospitality cover, and one fatal seam.

A lair should answer:
- what kind of place this is
- what social story it tells visitors
- how access is controlled
- what defensive logic makes it function
- what vulnerability ruins the mystique

## Why They Matter

Villain profiles generate character truth.
Schemes generate motion.
Lairs generate spatial truth.

Once both exist, the world graph stops being a profile network and starts being a franchise engine.

## Taxonomy Layer

The second artifact adds the named trope vocabulary layer the first pass was missing:
- plot goals
- plot-shapes / gambits
- betrayal modes
- reversal modes
- heist beats and crew roles
- lair features
- lair story-function logic
- satirical meta voice

## Spatial Grammar

The final lair artifact contributes one especially reusable spatial sequence:

`public mask -> threshold test -> ceremonial void -> operations ring -> private sanctum -> hidden escape route`

That grammar should sit beside the lair feature pool and story-function pool as a reusable architectural script.
