# Villain Pack Experiment Process

Villain Pack exists to prove that antagonist generation can be treated as a reusable compositional substrate rather than a one-off genre gimmick.

## Current loop

1. define villain families
2. define modifiers
3. define signifier packages
4. define dialogue grammars
5. define narrative payoff patterns
6. compose them into portable villain cards

## Why this matters

The point is not just “generate a cool bad guy.”
The point is to create a reusable engine that later systems can call into.

That means the first milestone is:
- structurally useful villain cards

Not yet:
- finished stories
- roleplay systems
- simulation loops
- polished UI
